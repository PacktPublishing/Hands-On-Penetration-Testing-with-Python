Full-Coverage Application Security Scanner :XTREME



Requirements :

  • Technology support : Tool provides support for web applications developed using following languages : Python, PHP, Visual Basic, Java, C#, Ruby, Perl.

  • Back end Support : The tool provides support for any of the above mentioned technologies using Basex XML Database engine at back end.Thus basex server needs to be installed on client machine that has web application to be tested.

  • Additional Requirements : The tool further requires Redis server ( key-value cache and store),to be installed and running on the client machine that has the web application to be tested.



Prerequisites for usage

BasexClient File replacement Steps :

The tool follows a black box approach towards detection of vulnerabilities in web applications and thus driver modification for each technology is required to be done for capturing the queries that are sent to the database server. BaseX client files were modified for the following web technologies with respect to BaseX server being used as the XML back end persistent store:

  • Python
  • PHP
  • Visual Basic
  • Java
  • C#
  • Ruby
  • Perl

Any web application using any the above mentioned technologies and Basex server at the backed could be used to test for vulnerabilities using this tool. However a slight replacement needs to be done:
The basexClient file of application under test needs to be replaced by the basexClient file provided here :
Following are the links to the modified driver files which could be downloaded and saved and then replaced with basexClient file of application to be tested.

  • Python   :  BasexClient.py
  • PHP   :  BasexClient.php
  • Visual Basic   :  BasexClient.vb
  • Java   :  BasexClient.java
  • C#   :  BasexClient.c#
  • Ruby   :  BasexClient.rb
  • Perl   :  BasexClient.pl

  • Step 1 : Start the Basex server


  • Step 2 : Start the Redis server


  • Step 3 : Start the web application server which is to be tested (suppose its running on http://localhost:9254/sample/index.jsp)


  • Step 4 : Go to the scan tab and click upon start new scanClick here


  • Step 5 : To begin xml injection scan click upon Xml injection Scan tab Click here


  • Step 6 : Now fill in all the required fields which will be needed to scan the web application and once all the fields are filled click upon start scanning button Click here

  • Step 8 :Here a user can view the detailed report of the tool.It contains the vulnerabilities and other details. .In order to view the details of each vulnerability a user can click upon the number beneath the # vulnerabilities tab.As shown in figure : Click here
