Start Url Example


  Seed url includes :

  • The protocol (http,https ,etc)
  • The domain name (localhost or etc):
  • The app name (if present)
  • The index page/start page

    • For eg :
  • Example 1 : http://localhost:8000/MYapp/index.jsp
  • Example 2 (Without app name) : http://www.mysite.com/index.php
Username Field example


  User name Field :

  • Specify the name/id of the form-element/text-field of your application that will have the user name /email /id or any other required credentials

  • If your app/wesbite is running ,you can right click on that field and from the opened menu go to inspect element option,that will read you the name or id of the field
Password Field example


  Password Field :

  • Specify the name/id of the form-element/text-field of your application that will have the Password or any other required credentials

  • If your app/wesbite is running ,you can right click on that field and from the opened menu go to inspect element option,that will read you the name or id of the field
Login URL


  • Specify the URL that contains the login form of the web application to be tested .
  • If your app/website is running ,you can go to the website and check the URL in the address bar
Logout URL


  • Specify the logout URL of your web application to be tested.
  • If your app/website is running ,you can go to the website and check the URL .
Additional Parameters


  • If your web application to be tested requires any additional authentication information apart from user name and password to login .For example :

  • Registration no :133 ,Role : admin and etc.
    A user needs to specify all additional authentication parameters and their values in this section (parameter name :parameter value ) in a comma separated manner

  • For example : Role : admin , DOB : 01-01-2002
    The shown example assumes that the login form apart from user name/email/id and password field contains two other text fields named as "Role" and the expected valid value is : "admin". The other expected parameter is "DOB" with expected valid value :"01-01-2002"

  • Therefore if any login form may have more then two authentication fields,the third one and the expected value of the third one is to be specified here

  • Note !.. If there are no additional parameters at the login form,a user may leave this field blank or may specify a default value as "none :none"

  • Note : Once the scan is completed (which is run in parallel),its very important for the user to clear off the contents of the config file.

Public IP example


  • Specify the ip address of the web application to be tested

  • If your app/wesbite is running on local machine ,you can specify the IP address as :localhost
URLexample


  • Specify the complete URL of the page which contains vulnerable parameter for Xpath injection.

  • For example : http:localhost:9254/sample/index.jsp
    Assuming that the index page has got a vulnerable text field that could be exploited by Xpath injection attack
URL Example


  • Specify the method of the form at the web page that contains the text-field that is vulnerable to X-path injection .

  • For example : index.jsp is the web page that has a text field which would be vulnerable to Xpath injection and has a Html form whose action method might be Get or POST,then a user needs to specify the exact action method of that Html form.
Vulnerable Parameter


  • Specify the name/id of the text field / parameter at the above given URL which would be vulnerable to X-path injection.

  • For example : index.jsp is the web page that has a text field with id/name as "product_search" which would be vulnerable to Xpath injection,then a user needs to specify "product_search" as input here
Vulnerable Parameter value


  • Specify the value that the vulnerable parameter would be expected to take in order to produce a valid match String .
Match String


  • Specify the match String which the web application under test is expected to produce once data is posted on the specified url with all the above given input fields and values.

  • For example : 1 result found
